ABSTRAK
Dinas Pendidikan Pemerintah Provinsi Jawa Timur merupakan instansi pemerintah yang bertanggung jawab atas semua hal yang berkaitan dengan pendidikan menengah atas di Jawa Timur. Produk yang kami kerjakan saat melakukan Kerja Praktik adalah PPDB, yaitu sistem penerimaan siswa baru untuk membantu calon peserta didik baru untuk memilih SMA tujuan. Penggunaan aplikasi yang sudah ada memiliki kebutuhan untuk peserta didik baru yang akan mendaftar ke SMA negeri di provinsi Jawa Timur.
Aplikasi dibuat dengan menggunakan framework CodeIgniter dan memiliki beberapa fitur, beberapa diantaranya adalah pendaftaran peserta didik, informasi sekolah PPDB, dan perangkingan. Kami diminta untuk menyiapkan arsitektur yang tepat agar sistem dapat diakses oleh masyarakat dengan kendala minimal mengingit. Sistem ini menggunakan metode load balancing dengan bantuan webserver NGINX, HAProxy, dan database MySQL.
Kata Kunci : Website, PPDB, NGINX, HAProxy

1.1. Latar Belakang

Saat ini dunia telah berkembang menjadi era digital. Semua layanan dapat disajikan secara online dengan adanya teknologi yang dapat dimanfaatkan untuk mempermudah manusia dalam melakukan aktivitas tersebut. Bila dibandingkan dengan yang dulu, kita membaca koran atau majalah dengan membelinya dari koran atau majalah. Kini, untuk membaca koran kita dapat membuka website atau aplikasi penyedia koran untuk membacanya. Website atau aplikasi tidak hanya terbatas untuk membaca koran. Dengan website ataupun aplikasi manusia seakan-akan dapat melakukan segala hal dalam satu tempat. Banyak sekali fungsi dari website atau aplikasi, sebagai contohnya pembayaran pajak, e-banking, pembelajaran, dan lain lain.

Setiap sekolah secara suatu daerah mengadakan penerimaan peserta didik baru (PPDB) untuk sekolah dari SD, SMP, dan SMA. Salah satu daerah yang mengadakan penerimaan peserta didik baru adalah Provinsi Jawa Timur. Sekolah di Jawa Timur sangatlah banyak dan perlu pendaftar untuk menuju tempat pendaftaran berbeda-beda. Ada yang jauh ada juga yang dekat. Oleh karena itu, dibutuhkan suatu sistem untuk mengurusi penerimaan peserta didik baru tersebut. Tentunya sistem untuk pandemi seperti sekarang ini, semua proses yang biasanya masih dapat dilakukan secara tatap muka kini dilakukan secara online. Penerimaan peserta didik baru yang kami kerjakan untuk jenjang SMA. Pada saat KP, kami diberi kesempatan untuk merancang dan mengimplementasikan infrastruktur dari sistem penerimaan peserta didik baru tersebut.

3.1. Pemrograman Web

Web atau World Wide Web adalah ruang informasi yang berisi dokumen dan resource web lainnya yang dapat diidentifikasi melalui sebuah URL (Uniform Resource Locators, contohnya www.google.com), dan ditakses ketika terkoneksi dengan internet. Halaman perpedia dokumen di dalam web dapat disebut sebagai website yang dapat terkoneksi satu dengan lainnya (hyperlink).

Pemrograman web adalah proses pembuatan halaman tersebut agar bisa diakses oleh semua orang. Dalam pembuatan website, diperlukan sebuah standar pada website agar semua orang dapat membaca informasi dalam keadaan yang berbeda. Standar tersebut adalah HTML (Hypertext Markup Language). Jadi pemrograman web memiliki tugas untuk menciptakan suatu halaman sesuai standar HTML agar semua orang memiliki akses pada informasi di dalam halaman tersebut.

3.2. HTML

Bahasa standar internasional yang digunakan untuk membuat halaman web. HTML menggambarkan struktur dan isi sebuah dari sebuah dokumen. HTML biasanya digabungkan dengan css dan javascript. css untuk mempermudah tampilan dan javascript untuk client side scripting language.

3.3. Javascript

Javascript adalah sebuah bahasa tingkat tinggi yang dinamis. Javascript memiliki banyak sekali fungsionalitas seperti web application, backend application, internet of things (IoT), dan lain - lain. Pada buku kerja praktik ini javascript digunakan untuk client side scripting language yang tertanam pada HTML sebuah website. Javascript juga memiliki banyak library yang dapat digunakan contohnya nodejs, axios, bluebirdjs, vuejs, angularjs, reactjs, animatejs, dan lain - lain.

3.4. TypeScript

TypeScript adalah bahasa pemrograman yang dikembangkan oleh Microsoft sebagai superset dari JavaScript. TypeScript menambahkan sistem tipe statis opsional ke JavaScript, yang memungkinkan pengembang untuk mendeteksi kesalahan lebih awal dalam proses pengembangan. TypeScript dikompilasi menjadi JavaScript biasa yang dapat dijalankan di mana saja JavaScript dapat berjalan. Dalam proyek ini, TypeScript digunakan untuk pengembangan frontend dengan React dan Next.js untuk meningkatkan keamanan tipe dan produktivitas pengembangan.

3.5. SQL Server

SQL Server adalah sistem manajemen basis data relasional (RDBMS) yang dikembangkan oleh Microsoft. SQL Server digunakan untuk menyimpan dan mengelola data aplikasi dengan dukungan penuh untuk operasi CRUD (Create, Read, Update, Delete). SQL Server menyediakan fitur-fitur enterprise seperti backup otomatis, replikasi, dan keamanan tingkat tinggi. Dalam sistem manajemen inventori ini, SQL Server digunakan untuk menyimpan data barang, transaksi, supplier, dan data master lainnya dengan performa dan keandalan yang tinggi.

7
3.6. Web Server (Nginx)

Salah satu komponen penting di dalam website adalah web server. Web server berfungsi sebagai penerima request dari browser yang kemudian memberikan tanggapan dengan mengirimkan halaman situs web dalam bentuk dokumen HTML. Nginx adalah salah satu web server populer saat ini. Selain memberikan performa yang andal, Nginx juga mempunyai beberapa fitur canggih lain yang mudah dikonfigurasi. Jadi tentu saja akan membuat website Anda lebih powerful dan canggih.

Nginx menawarkan penggunaan konkurensi tinggi yang dapat menangani ribuan koneksi secara bersamaan dengan penggunaan memori yang rendah. Nginx menggunakan proses asynchronous (mengeksekusi kode sebelum perintah sebelumnya selesai dijalankan) untuk membuat proses baru untuk setiap eksekusi baris kode yang ada. Tidak hanya memudahkan proses asynchronous, Nginx juga melakukan pendekatan event-driven. Pendekatan event-driven merupakan sebuah model (permintaan atau proses) di dalam satu thread saja. Jadi Nginx tidak membuat proses baru ketika ada permintaan (web request), tapi ditangani di dalam satu thread (asynchronous dan pendekatan event-driven). Melalui berbagai macam teknologi ini Nginx merevolusi bagaimana web server bekerja dan berhasil menjadi web server paling populer sampai dengan saat ini.

3.7. HAProxy

HAProxy adalah sebuah software open source yang menyediakan load balancer dengan ketersediaan

8

tinggi dan proxy server untuk aplikasi berbasis TCP dan HTTP yang tersebar pada banyak server. HAProxy membantu membagi beban request database ke beberapa server berbeda untuk menghindari terjadinya kelebihan request pada database.

3.8. React.js

React.js adalah library JavaScript yang digunakan untuk membangun user interface (UI) aplikasi web. React dikembangkan oleh Facebook dan menjadi salah satu teknologi frontend yang paling populer. React menggunakan konsep component-based architecture yang memungkinkan pengembang untuk membuat UI yang dapat digunakan kembali dan mudah dikelola.

3.9. Next.js

Next.js adalah framework React yang menyediakan fitur-fitur tambahan seperti server-side rendering (SSR), static site generation (SSG), dan routing otomatis. Next.js memungkinkan pengembangan aplikasi web yang lebih cepat dan SEO-friendly dengan performa yang optimal.

3.10. ASP.NET Core

ASP.NET Core adalah framework web modern yang dikembangkan oleh Microsoft untuk membangun aplikasi web dan API. Framework ini bersifat cross-platform, open-source, dan memiliki performa tinggi. ASP.NET Core mendukung arsitektur MVC (Model-View-Controller) dan Web API untuk pengembangan aplikasi backend yang scalable.

3.11. Entity Framework Core

Entity Framework Core adalah Object-Relational Mapping (ORM) framework yang memungkinkan pengembang untuk bekerja dengan database menggunakan objek .NET. EF Core menyederhanakan akses data dengan menyediakan abstraksi tingkat tinggi untuk operasi database seperti CRUD operations.

BAB IV
ANALISIS DAN PERANCANGAN INFRASTRUKTUR
SISTEM
4.1. Analisis Sistem

Pada bab ini akan dijelaskan mengenai tahapan dalam membangun infrastruktur aplikasi sistem manajemen inventori yaitu analisis dari infrastruktur sistem yang akan dibangun. Hal tersebut dijelaskan ke dalam dua bagian, definisi umum aplikasi dan analisis kebutuhan.

4.1.1. Definisi Umum Aplikasi

Secara umum, aplikasi sistem manajemen inventori merupakan sistem pengelolaan stok barang berbasis website yang digunakan untuk mengelola dan memantau inventori perusahaan. Sistem ini memiliki dua view dari dua sisi pengguna yaitu input data barang (aplikasi admin) dan laporan (aplikasi user dan admin).

4.2. Perancangan Infrastruktur Sistem
4.2.1. Desain Sistem

Desain arsitektur pada Sistem Manajemen Inventori PT Petronika ini dilakukan pembagian beban untuk web server load balancing dengan NGINX, yaitu menggunakan satu server sebagai server gateway load balancing. Server frontend di-deploy ke platform Vercel untuk skalabilitas dan performa optimal, sedangkan backend menggunakan ASP.NET Core dengan .NET runtime yang di-deploy pada server fisik. Database SQL Server juga berada pada server yang sama dengan backend untuk meminimalkan latency. NGINX bertugas sebagai reverse proxy dan load balancer untuk menangani request dari frontend ke backend API. Desain server clustering tersebut dapat dilihat pada Gambar 1.

10
Gambar 4.1. Desain Arsitektur Server

Gambar 4.1 merupakan desain arsitektur server clustering yang digunakan pada Sistem Manajemen Inventori PT Petronika 2024. Frontend aplikasi di-deploy ke platform Vercel yang menyediakan CDN global dan auto-scaling. Backend API menggunakan ASP.NET Core dengan .NET runtime yang di-deploy pada server fisik bersama dengan database SQL Server untuk meminimalkan latency komunikasi database.

NGINX bertugas sebagai reverse proxy dan load balancer yang menangani request dari frontend ke backend API. Sistem ini dirancang untuk menangani beban kerja yang tinggi dengan distribusi load yang optimal untuk memastikan performa dan availability sistem yang maksimal.

BAB VII

KESIMPULAN DAN SARAN

7.1. Kesimpulan

Kesimpulan yang didapat setelah melakukan perancangan dan implementasi sistem aplikasi inventory management untuk PT. Shopee adalah sebagai berikut:

a. Sistem inventory management yang dibangun telah berhasil mengimplementasikan arsitektur modern dengan menggunakan Next.js 15 untuk frontend dan ASP.NET Core 8.0 untuk backend, memberikan performa yang optimal dan pengalaman pengguna yang responsif.

b. Implementasi fitur-fitur utama seperti manajemen raw materials, finished products, suppliers, customers, dan stock transactions telah berhasil dilakukan dengan menggunakan Entity Framework Core sebagai ORM dan SQL Server sebagai database management system.

c. Sistem autentikasi menggunakan JWT (JSON Web Token) telah berhasil diimplementasikan dengan baik, memberikan keamanan yang memadai untuk akses ke sistem inventory management.

d. Dashboard yang informatif telah berhasil dikembangkan dengan menampilkan statistik real-time mengenai total products, suppliers, customers, dan recent transactions, memudahkan manajemen dalam monitoring inventory.

e. Sistem reporting yang komprehensif telah berhasil diimplementasikan, memungkinkan pengguna untuk menganalisis data inventory dengan lebih efektif melalui berbagai laporan yang tersedia.

7.2. Saran

Saran untuk pengembangan lebih lanjut sistem inventory management ini adalah sebagai berikut:

a. Implementasi fitur notifikasi real-time menggunakan SignalR untuk memberikan update langsung kepada pengguna ketika terjadi perubahan stock atau transaksi penting.

b. Penambahan fitur forecasting dan analytics menggunakan machine learning untuk memprediksi kebutuhan inventory berdasarkan historical data dan trend pasar.

c. Implementasi mobile application menggunakan React Native atau Flutter untuk memberikan akses yang lebih fleksibel kepada pengguna dalam mengelola inventory dari perangkat mobile.

d. Penambahan fitur barcode scanning dan QR code untuk mempercepat proses input dan tracking inventory items.

e. Implementasi integration dengan sistem ERP existing perusahaan untuk sinkronisasi data yang lebih seamless dan menghindari duplikasi data entry.

f. Penambahan fitur audit trail yang lebih detail untuk tracking semua perubahan data dan meningkatkan transparency dalam operasional inventory management.
