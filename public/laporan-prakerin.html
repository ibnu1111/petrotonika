BAB I
PENDAHULUAN

1.1 Latar Belakang

Perkembangan teknologi informasi yang pesat telah mengubah cara perusahaan dalam mengelola operasional bisnis mereka. Sistem manajemen inventori merupakan salah satu aspek penting dalam operasional perusahaan, terutama untuk perusahaan manufaktur seperti PT Petronika yang bergerak di bidang industri kimia.

PT Petronika sebagai perusahaan yang memproduksi bahan kimia seperti DOP (Dioctyl Phthalate) dan DINP (Diisononyl Phthalate) memerlukan sistem yang efisien untuk mengelola inventori bahan baku dan produk jadi. Pengelolaan inventori yang masih manual dapat menyebabkan berbagai masalah seperti:

  1. Kesulitan dalam tracking stok real-time
  2. Human error dalam pencatatan
  3. Ineffisiensi dalam perencanaan produksi
  4. Kesulitan dalam pembuatan laporan

Berdasarkan permasalahan tersebut, dikembangkanlah sistem manajemen inventori berbasis web yang dapat membantu PT Petronika dalam mengelola inventori secara lebih efisien dan akurat.

1.2 Rumusan Masalah

Berdasarkan latar belakang di atas, rumusan masalah dalam pengembangan sistem ini adalah:

  1. Bagaimana merancang sistem manajemen inventori yang sesuai dengan kebutuhan PT Petronika?
  2. Bagaimana mengimplementasikan sistem manajemen inventori berbasis web menggunakan teknologi modern?
  3. Bagaimana sistem dapat memberikan informasi real-time mengenai status inventori?
  4. Bagaimana sistem dapat menghasilkan laporan yang komprehensif untuk mendukung pengambilan keputusan?

1.3 Tujuan

Tujuan dari pengembangan sistem manajemen inventori ini adalah:

Tujuan Umum:

  • Mengembangkan sistem manajemen inventori berbasis web untuk PT Petronika
  • Meningkatkan efisiensi pengelolaan inventori perusahaan

Tujuan Khusus:

  • Merancang database yang optimal untuk menyimpan data inventori
  • Mengimplementasikan fitur CRUD (Create, Read, Update, Delete) untuk manajemen data
  • Membuat sistem tracking stok real-time
  • Mengembangkan fitur pelaporan yang komprehensif
  • Membuat interface yang user-friendly dan responsive

1.4 Manfaat

Manfaat yang diharapkan dari pengembangan sistem ini adalah:

Bagi Perusahaan:

  • Meningkatkan akurasi data inventori
  • Mengurangi human error dalam pencatatan
  • Mempercepat proses pelaporan
  • Mendukung pengambilan keputusan yang lebih baik

Bagi Siswa:

  • Mendapatkan pengalaman praktis dalam pengembangan sistem
  • Meningkatkan kemampuan programming dan problem solving
  • Memahami kebutuhan industri dalam bidang IT

1.5 Batasan Masalah

Untuk memfokuskan pengembangan sistem, batasan masalah yang ditetapkan adalah:

  1. Sistem difokuskan pada manajemen inventori bahan baku dan produk jadi
  2. Sistem tidak mencakup modul accounting/keuangan
  3. Sistem dikembangkan berbasis web dengan teknologi Next.js dan ASP.NET Core
  4. Implementasi dibatasi pada lingkungan PT Petronika

4.2 Desain Sistem

4.2.1 Arsitektur Sistem

Sistem menggunakan arsitektur 3-Tier Architecture:

1. Presentation Layer (Frontend)

  • Next.js Application dengan TypeScript
  • Responsive UI menggunakan Tailwind CSS
  • Client-side state management dengan React Context

2. Business Logic Layer (Backend)

  • ASP.NET Core Web API
  • Business logic dan validation rules
  • JWT-based authentication
  • RESTful API design

3. Data Access Layer (Database)

  • SQL Server database
  • Entity Framework Core sebagai ORM
  • Database migrations untuk version control
  • Stored procedures untuk complex queries

4.2.2 Database Design

Tabel Utama:

1. Users

- Id (int, PK, Identity) - Username (nvarchar(50), Unique) - Password (nvarchar(255)) - Name (nvarchar(100)) - Role (nvarchar(20)) - IsActive (bit) - CreatedAt (datetime2) - UpdatedAt (datetime2)

2. Suppliers

- Id (int, PK, Identity) - Name (nvarchar(100)) - Code (nvarchar(20), Unique) - Contact (nvarchar(100)) - Phone (nvarchar(20)) - Email (nvarchar(100)) - IsActive (bit) - CreatedAt (datetime2) - UpdatedAt (datetime2)

3. RawMaterials

- Id (int, PK, Identity) - Name (nvarchar(100)) - Code (nvarchar(20), Unique) - Description (nvarchar(500)) - Unit (nvarchar(20)) - CurrentStock (decimal(18,2)) - MinimumStock (decimal(18,2)) - Price (decimal(18,2)) - SupplierId (int, FK to Suppliers) - CreatedAt (datetime2) - UpdatedAt (datetime2)

4. FinishedProducts

- Id (int, PK, Identity) - Name (nvarchar(100)) - Code (nvarchar(20), Unique) - Description (nvarchar(500)) - Unit (nvarchar(20)) - CurrentStock (decimal(18,2)) - MinimumStock (decimal(18,2)) - Price (decimal(18,2)) - CreatedAt (datetime2) - UpdatedAt (datetime2)

5. StockTransactions

- Id (int, PK, Identity) - ItemId (int) - ItemType (nvarchar(20)) -- 'RAW_MATERIAL' atau 'FINISHED_PRODUCT' - TransactionType (nvarchar(10)) -- 'IN' atau 'OUT' - Quantity (decimal(18,2)) - Unit (nvarchar(20)) - Reference (nvarchar(50)) - Notes (nvarchar(500)) - UserId (int, FK to Users) - CreatedAt (datetime2)

Dashboard Endpoints:

[HttpPost("login")] public async Task<IActionResult> Login([FromBody] LoginRequest request) { var user = await _context.Users .FirstOrDefaultAsync(u => u.Username == request.Username && u.IsActive); if (user == null || !BCrypt.Net.BCrypt.Verify(request.Password, user.Password)) { return Unauthorized(new { success = false, message = "Invalid credentials" }); } var token = GenerateJwtToken(user); return Ok(new { success = true, data = new { token, user = new { user.Id, user.Username, user.Name, user.Role } } }); }

CRUD Operations dengan Entity Framework:
